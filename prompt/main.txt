I want to build a mobile app with the name "Cidy" which allows the government teachers of Tunisia who run private classes to : 
    - track the attendance of their students into their classes
    - track the payments of their students
    - have notifications about the the due payment of students 
    - have a clear idea on their revenue and the due payments of their students
    - have a clear idea about their weekly schedule
    - send due payment notifications to the student and his parent
    - give the ability to the parents of the students to view the attendance and the payments of their sons so that the son
      can't fool his parents about their payments and attendance
    - give the ability for the students to view their attendance and payments 

- the app does that throught having 3 profiles a teacher,a student and a parent and common login and register screens: 

- the description for the common Login and Register screens :
    - The common login screen:
        - UI:
            - the logo of the app Cidy 
            - A text input for the email (required, email format)
            - A password input for the password with the ability to show and hide the content of the input (required)
            - A button with the name "Login"
            - A clickable text with the content: "Don't have an account? Register here", which routes to the register screen
        - Dynamics:
            - The "Login" button authenticates the user using the provided email and password. Upon successful authentication, the user is routed to their respective dashboard based on their profile type (teacher, student, or parent).

    - The common register screen:
        - UI:
            - the logo of the app Cidy
            - A select input for the profile type with the following options:
                - Teacher
                - Student
                - Parent
            - A text input for the full name (required)
            - A text input for the email (required, email format)
            - A text input for the phone number (required, phone number format, min length = max length = 8)
            - A password input for the password with the ability to show and hide the content of the input (required, min length = 6, contains at least one uppercase letter, one lowercase letter, and one number)
            - A password input to confirm the password with the ability to show and hide the content of the input (required, must match the password)
            - A button with the name "Register"
            - A clickable text with the content: "Already have an account? Login here", which routes to the login screen
        - Dynamics:
            - The "Register" button validates the inputs and creates a new account for the selected profile type. Upon successful registration, the user is routed to their respective dashboard based on their profile type (teacher, student, or parent).
